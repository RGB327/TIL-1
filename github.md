# 깃허브 사용법
-----
## 깃의 명령어들

### 초기 설정 명령어
- **git init**
    - 현재 폴더를 Git 저장소로 초기화함
- **git clone <깃허브 링크>**
    - GitHub 저장소를 복제해서 로컬에 가져옴
---
### 저장 및 기록 명령어
- **git add <파일명>**
    - 변경된 파일을 스테이징 영역으로 올림 (파일명에 . 입력 시 변경된 모든 파일 전체 선택)
- **git commit -m**
    - add를 통해 스테이징 영역으로 올린 코드들을 로컬 저장소에 메세지와 함께 확정
- **git status**
    - 현재 변경 상태 확인(수정/추가/커밋 여부)
- **git log**
    - 지금까지의 커밋 기록 확인
---
### 원격 저장소(Github) 연결 명령어
- **git push**
    - 로컬 커밋을 깃허브에 올림
- **git pull**
    - 깃허브의 변경 내용을 가져와 병합함
- **git fetch**
    - 깃허브에서 변경 내용을 가져오기만 함(병합x)
### 되돌리기 명령어

- **git reset --(hard or soft) (commit-id)**
    - hard는 특정 커밋 시점으로 돌아가며 그 이후 있었던 모든 변경사항을 삭제함
    - soft는 삭제한 변경 내용은 스테이징 영역에 남겨둠
- **git revert (commit-id)**
    - 해당 커밋의 변경사항을 취소하는 새로운 커밋을 생성
- **git restore --staged (파일명)**
    - add를 취소함 / . 쓰면 전체 되돌리기

### branch 관련 명령어
- **git branch**
    - 브랜치 목록 확인
- **git checkout (브랜치명)**
    - 해당 브랜치로 이동 / checkout -b 사용시 생성하고 이동
